## 项目主要需求

**项目概述**：剧本杀门店运营管理系统 —— 面向剧本杀线下门店，提供剧本库管理、场次排期、玩家组局、DM排班和会员运营等一站式门店运营工具。

**项目类型**：全栈Web应用

**功能需求**：
1. **剧本库与DM管理**：录入剧本信息（名称、类型、难度、时长、人数、主持人DM要求），上传剧本封面与简介，关联专属DM主持人，支持按标签筛选与搜索。
2. **场次排期与拼车位**：门店设置每日开放场次时段（如14:00场、19:00场），每场显示已报名人数和剩余空位，玩家可单人报名加入拼车位或自行组满局。
3. **玩家组局与角色分配**：满局后DM可为玩家分配角色，支持随机分配与手动调整，系统记录每次组局玩家名单与角色分配结果。
4. **会员积分与等级体系**：注册会员消费积累积分，设置等级规则（如青铜/白银/黄金/钻石），不同等级享受折扣与优先拼车位权益，积分可兑换周边或抵扣费用。
5. **营收与上座率分析**：管理员查看每日/周/月营收报表、各剧本上座率排行、DM带本场次与评分统计，支持导出营业数据。

**技术要求**：
- 英文短名（用于 Docker Compose）：`ldmurdergame`
- 前端端口：`28502`，后端端口：`29502`
- 前端框架：Vue 3 + TypeScript
- UI 库：Element Plus
- 构建工具：Vite
- 后端框架：Spring Boot + Java
- 数据库：PostgreSQL
- 认证方式：JWT
- 其他依赖：MyBatis、Maven

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=ldmurdergame`，确保 Compose 项目名不依赖目录名（中文目录名会被规范化为空导致 `project name must not be empty` 错误）
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**（Compose V2 已废弃，会产生 `the attribute version is obsolete` 警告）
- 顶层声明 `name: ldmurdergame` 作为兜底，避免目录名为中文时项目名为空
- 所有服务使用 `container_name` 命名容器，便于管理；容器名建议带 `${COMPOSE_PROJECT_NAME:-ldmurdergame}` 前缀
- 数据库数据使用命名卷持久化（`volumes:` 顶层声明，不要使用绑定挂载到中文路径）
- 前端通过 Nginx 反向代理后端 API（前端代码里调用 `/api`，由 Nginx 转发到 `backend:29502`，不要硬编码 `localhost`）
- 服务间通过 Docker 内部网络通信，服务名即主机名
- 数据库服务配置 `healthcheck`，后端服务通过 `depends_on: { db: { condition: service_healthy } }` 等待数据库就绪
- 后端服务也建议配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `80:80`，后端按框架默认端口，数据库使用标准端口
- 所有可变配置（端口、数据库账号密码、JWT 密钥等）通过环境变量注入，并在 `.env.example` 中给出默认值
- 支持通过 `docker compose up -d` 一条命令启动全部服务，不依赖额外的 `-p` 参数

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME`：英文短名（小写字母/数字/`-`/`_`），如 `ldmurdergame`
- 数据库连接相关：`DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_ROOT_PASSWORD`
- 后端：`JWT_SECRET` 或等价的密钥配置
- 端口映射：`FRONTEND_PORT`、`BACKEND_PORT`、`DB_PORT`（可选，便于本地冲突时调整）

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:29502/`
- 配置 `try_files $uri $uri/ /index.html;` 支持前端 SPA 路由
- WebSocket 场景需配置 `Upgrade` / `Connection` 头

自检清单（生成代码后必须确认）：
1. 在 **中文目录名** 下执行 `docker compose config --quiet` 不报错
2. 启动后 `docker compose ps` 显示全部容器 healthy
3. 浏览器访问前端端口能看到页面，且 `/api/...` 请求返回正常
4. README 中的 Docker 启动命令是 `docker compose up -d`（不需要 `-p`）

## docker-compose.yml 参考骨架（务必遵循）

```yaml
name: ldmurdergame   # 顶层 name 字段兜底，避免中文目录名导致 project name 为空

services:
  db:
    image: postgres:15
    container_name: ${COMPOSE_PROJECT_NAME:-ldmurdergame}-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${DB_NAME}
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER} -d ${DB_NAME}"]
      interval: 5s
      timeout: 5s
      retries: 20

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-ldmurdergame}-backend
    restart: unless-stopped
    environment:
      DB_HOST: db
      DB_PORT: 5432
      DB_NAME: ${DB_NAME}
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-ldmurdergame}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-80}:80"
    depends_on:
      - backend

volumes:
  db_data:
```
